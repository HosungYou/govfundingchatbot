## 1. Release Summary
The 1.1.0 release reorganizes the repository around modular application packages, introduces architecture/operations documentation, and formalizes agent responsibilities. It sets the stage for future services (API, alerts, web) while keeping the ETL pipeline intact under its new namespace.

## 2. Release Type
**Minor Release** – structural improvements, additional documentation, and developer workflow updates without breaking runtime behavior.

## 3. Change Highlights
- **Repository Architecture**
  - Moved ETL implementation to `apps/etl/` with package exports so future services can sit side-by-side (`apps/__init__.py`, `apps/etl/*`).
  - Updated Poetry configuration and scripts to reference the new module path, ensuring `scripts/run_etl.sh` executes `python -m apps.etl.pipeline`.
- **Documentation Overhaul**
  - Authored `PROJECT_STRUCTURE.md`, `docs/architecture/system-overview.md`, and `docs/backlog/unimplemented-features.md` to clarify system design and outstanding work.
  - Logged stakeholder requests in `docs/requests/2025-10-10.md` and defined agent responsibilities in `Agents.md`.
  - Refreshed `README.md` to reflect the reorganized layout and upcoming roadmap.
- **Release Management**
  - Established release-notes scaffolding (`release-notes/README.md`) and documented v1.0.0 foundations for future tagging.

## 4. Technical Details & Code Samples
```bash
# scripts/run_etl.sh
poetry run python -m apps.etl.pipeline "$@"
```
- The ETL entrypoint now resolves through the `apps` package, aligning with the reorganized module hierarchy.
- `pyproject.toml` declares `packages = [{ include = "apps" }]` so Poetry exports the new namespace when building distributions.
- Architecture documentation includes a Mermaid diagram showing data ingestion → storage → services flow (`docs/architecture/system-overview.md`).

## 5. Breaking Changes
- **Module Path Updates**: Any external automation calling `python -m etl.pipeline` must switch to `python -m apps.etl.pipeline`.
- **Documentation Paths**: Product specs moved under `docs/product/`; update bookmarks and references accordingly.

## 6. Migration & Upgrade Notes
1. Pull the latest main branch: `git pull origin main`.
2. Reinstall dependencies to pick up the new package layout: `poetry install`.
3. Update scheduled jobs or notebooks to execute `scripts/run_etl.sh` (or `python -m apps.etl.pipeline`).
4. Review the new architecture/backlog docs to align ongoing workstreams.

## 7. Deployment Checklist
- [ ] Update CI/CD workflows to call the new script path.
- [ ] Ensure Supabase credentials in `.env` are compatible with upcoming API service work.
- [ ] Share the `PROJECT_STRUCTURE.md` overview with contributors.
- [ ] Plan infra directory creation for migrations/terraform in the next release.

## 8. Known Issues & Limitations
- API, alerting, and web application services remain unimplemented (tracked in `docs/backlog/unimplemented-features.md`).
- ETL still lacks retry/backoff and Supabase integration; future work will address robustness.
