# Release Notes v1.1.1 - AI Chat Integration

**Release Date:** October 10, 2025
**Version:** 1.1.1
**Type:** Feature Release

---

## 🎯 Overview

This release introduces **conversational AI capabilities** directly into the GovFunding platform, enabling users to discover federal funding opportunities through natural language interactions. The AI Chat feature provides personalized grant recommendations, answers eligibility questions, and helps researchers navigate the complex federal funding landscape.

---

## ✨ What's New

### 1. **Floating AI Chat Interface**

A persistent, always-accessible chat button appears on every page, providing instant access to our AI-powered grant assistant.

**Key Features:**
- **Global Accessibility**: Chat button available on all pages (landing, dashboard, search, detail pages)
- **Minimalist Design**: Non-intrusive floating button in bottom-right corner
- **Smooth Animations**: Elegant transitions and hover effects
- **Mobile Responsive**: Optimized for desktop and mobile devices

**User Experience:**
- Click the floating chat button to open/close the chat panel
- Clean, modern UI with gradient header
- Real-time streaming responses from GPT-4
- Auto-scrolling to latest messages
- Typing indicators during AI processing

### 2. **Intelligent Grant Search via Conversation**

Users can now find grants using natural language instead of rigid keyword searches.

**Capabilities:**
- **Semantic Understanding**: AI interprets research intent beyond exact keywords
- **Contextual Recommendations**: Uses Retrieval-Augmented Generation (RAG) to find relevant opportunities
- **Multi-turn Conversations**: Remembers context across multiple questions
- **Rich Responses**: Formatted answers with grant titles, agencies, deadlines, and funding amounts

**Example Conversations:**
```
User: "What are the latest NSF opportunities for AI research?"
AI: Provides 3-5 relevant grants with details and direct links

User: "Find grants for machine learning in healthcare"
AI: Searches across agencies and returns matching opportunities

User: "Explain eligibility requirements for NSF CAREER awards"
AI: Provides detailed explanation based on grant database
```

### 3. **Dashboard Chat History**

All conversations are automatically saved and displayed on the dashboard for easy review and continuation.

**Features:**
- **Persistent Storage**: Conversations saved in browser localStorage
- **Session Management**: Up to 10 most recent conversations stored
- **Expandable View**: Click any conversation to view full message history
- **Delete Capability**: Remove unwanted conversations
- **Real-time Updates**: Dashboard automatically refreshes when new messages are sent
- **Metadata Display**: Shows message count, date, and time for each session

**Dashboard Layout:**

## 🏗️ Technical Implementation

### New Components

#### `FloatingChat.tsx` (Client Component)
- **Purpose**: Main chat interface with floating button
- **State Management**: React hooks for messages, sessions, loading states
- **API Integration**: Streams responses from `/api/chat` endpoint
- **Persistence**: localStorage for cross-session history
- **Features**:
  - Real-time streaming with ReadableStream API
  - Auto-scroll to latest messages
  - Session creation and management
  - Error handling with user-friendly messages

#### `ChatHistory.tsx` (Client Component)
- **Purpose**: Dashboard widget displaying conversation history
- **Features**:
  - Real-time updates via storage events
  - Polling mechanism for same-tab updates (2s interval)
  - Expandable conversation threads
  - Delete functionality
  - Empty state with call-to-action

### Updated Files

#### `apps/web/app/layout.tsx`
- Added `<FloatingChat />` component to root layout
- Chat now available globally across all pages

#### `apps/web/app/dashboard/page.tsx`
- Integrated `<ChatHistory />` component
- Reorganized layout with chat history section
- Updated page structure for better UX

#### `apps/web/app/api/chat/route.ts` (v1.1.0 → v1.1.1)
- **Fixed**: TypeScript streaming error with OpenAI SDK v4
- **Changed**: Replaced `OpenAIStream` from 'ai' package with native `ReadableStream`
- **Improved**: Better error handling and edge runtime compatibility

### Data Flow

```
User Input (FloatingChat)
    ↓
POST /api/chat
    ↓
OpenAI GPT-4 + RAG Context (Supabase search)
    ↓
Streaming Response (ReadableStream)
    ↓
Real-time UI Update
    ↓
localStorage Persistence
    ↓
Dashboard ChatHistory Display
```

### Storage Schema

**localStorage Key**: `chat_sessions`

**Data Structure**:
```typescript
interface ChatSession {
  id: string              // "session_1728576000000"
  title: string           // First message preview (50 chars)
  messages: Message[]     // Full conversation thread
  createdAt: number       // Unix timestamp
  updatedAt: number       // Unix timestamp
}

interface Message {
  id: string              // "msg_1728576123456"
  role: 'user' | 'assistant'
  content: string         // Message text
  timestamp: number       // Unix timestamp
}
```

## 🐛 Bug Fixes

### Critical Fixes

**TypeScript Streaming Error (v1.1.0 → v1.1.1)**
- **Issue**: Vercel build failing with type mismatch on `OpenAIStream(response)`
- **Error**: `Type 'Stream<ChatCompletionChunk>' is not assignable to parameter of type 'Response | AsyncIterableOpenAIStreamReturnTypes'`
- **Root Cause**: Incompatibility between OpenAI SDK v4 and 'ai' package v3.0.0
- **Solution**: Replaced `OpenAIStream`/`StreamingTextResponse` with native `ReadableStream` implementation
- **Impact**: Build now succeeds, deployment works on Vercel Edge Runtime
- **Files Changed**: `apps/web/app/api/chat/route.ts`

## 🔐 Security & Privacy

### Data Handling
- **Client-side Storage**: Conversations stored in browser localStorage (not server)
- **User Privacy**: No personal data sent to OpenAI (only grant queries)
- **API Security**: Environment variables for API keys, never exposed to client
- **Rate Limiting**: Consider implementing per-user rate limits (future)

### Compliance
- **GDPR**: Users can delete their own localStorage data
- **Data Retention**: Browser-controlled (user can clear storage anytime)
- **Third-party APIs**: OpenAI (GPT-4), Supabase (grant data)
